52/**
53 * <p>
54 * This class implements the LZF lossless data compression algorithm. LZF is a
55 * Lempel-Ziv variant with byte-aligned output, and optimized for speed.
56 * </p>
57 * <p>
58 * Safety/Use Notes:
59 * </p>
60 * <ul>
61 * <li>Each instance should be used by a single thread only.</li>
62 * <li>The data buffers should be smaller than 1 GB.</li>
63 * <li>For performance reasons, safety checks on expansion are omitted.</li>
64 * <li>Invalid compressed data can cause an ArrayIndexOutOfBoundsException.</li>
65 * </ul>
66 * <p>
67 * The LZF compressed format knows literal runs and back-references:
68 * </p>
69 * <ul>
70 * <li>Literal run: directly copy bytes from input to output.</li>
71 * <li>Back-reference: copy previous data to output stream, with specified
72 * offset from location and length. The length is at least 3 bytes.</li>
73 * </ul>
74 *<p>
75 * The first byte of the compressed stream is the control byte. For literal
76 * runs, the highest three bits of the control byte are not set, the the lower
77 * bits are the literal run length, and the next bytes are data to copy directly
78 * into the output. For back-references, the highest three bits of the control
79 * byte are the back-reference length. If all three bits are set, then the
80 * back-reference length is stored in the next byte. The lower bits of the
81 * control byte combined with the next byte form the offset for the
82 * back-reference.
83 * </p>
84 */
